\begin{abstract}
This project implements a complete customer churn analytics system in two milestones. Milestone 1 delivers a machine learning pipeline for telecom churn prediction using classical supervised learning, preprocessing, model evaluation, visualization, and an interactive Gradio interface. Milestone 2 extends the same system into an agentic retention strategist built with LangGraph, local retrieval using FAISS, and Gemini-based structured generation. The final system predicts churn risk, explains the major churn drivers, retrieves grounded retention playbooks, and generates actionable intervention plans with follow-up question answering.
\end{abstract}

\section{Problem Statement}
Customer churn causes direct revenue loss and increases acquisition cost for telecom businesses. The objective of this project is to:
\begin{itemize}
    \item predict whether a customer is likely to churn,
    \item identify the main business drivers of that risk,
    \item present both single-customer and batch predictions through a hosted application,
    \item extend the prediction system into an AI assistant that proposes retention actions.
\end{itemize}

\section{Dataset Description}
The project uses the Telco Customer Churn dataset containing 7,043 customer records. Each row represents one customer and includes demographic, service, and billing information.

Important numeric features:
\begin{itemize}
    \item tenure
    \item MonthlyCharges
    \item TotalCharges
\end{itemize}

Important categorical features:
\begin{itemize}
    \item gender
    \item Partner
    \item Dependents
    \item InternetService
    \item Contract
    \item PaymentMethod
    \item TechSupport
    \item OnlineSecurity
\end{itemize}

The target variable \texttt{Churn} is encoded as:
\begin{itemize}
    \item 1 for churn
    \item 0 for retained
\end{itemize}

\section{Milestone 1: ML-Based Churn Prediction}

\subsection{Data Cleaning and Preprocessing}
The preprocessing pipeline is implemented using \texttt{ColumnTransformer}.

Cleaning steps:
\begin{itemize}
    \item convert \texttt{TotalCharges} to numeric,
    \item impute missing numeric values,
    \item remove the customer identifier,
    \item encode the churn label,
    \item standardize numeric features,
    \item one-hot encode categorical features.
\end{itemize}

\subsection{Exploratory Data Analysis}
The EDA module performs:
\begin{itemize}
    \item missing-value checks,
    \item churn distribution analysis,
    \item skewness checks,
    \item outlier detection,
    \item multicollinearity analysis using VIF,
    \item correlation heatmap generation.
\end{itemize}

\subsection{Model Development}
Two production models are used in the hosted system:
\begin{itemize}
    \item Logistic Regression
    \item Decision Tree
\end{itemize}

The Logistic Regression pipeline includes polynomial interaction features, Lasso-based feature selection, and SMOTE balancing. The Decision Tree pipeline uses preprocessing, SMOTE, and tuned tree hyperparameters.

\subsection{Evaluation Metrics}
The models are evaluated using:
\begin{itemize}
    \item Accuracy
    \item Precision
    \item Recall
    \item F1 Score
    \item Confusion Matrix
\end{itemize}

\section{Milestone 1 Interface}
The hosted Gradio application provides:
\begin{itemize}
    \item model performance dashboard,
    \item EDA visualizations,
    \item single-customer prediction form,
    \item CSV batch scoring with downloadable results,
    \item top churn-driver summaries.
\end{itemize}

\section{Milestone 2: Agentic Retention Strategist}

\subsection{System Overview}
Milestone 2 extends the prediction system into a stateful agent that turns churn risk into retention recommendations. The workflow is implemented with LangGraph and uses a local knowledge base plus retrieval.

\subsection{Agent Workflow}
The LangGraph flow contains the following stages:
\begin{enumerate}
    \item prepare customer input,
    \item score churn risk using the shared inference layer,
    \item summarize churn drivers and business context,
    \item retrieve relevant retention strategies from FAISS,
    \item generate a structured retention report using Gemini,
    \item answer follow-up questions using the same case state.
\end{enumerate}

\subsection{Explicit State}
The graph stores:
\begin{itemize}
    \item customer payload,
    \item chosen model,
    \item prediction result,
    \item risk summary,
    \item retrieval query,
    \item retrieved evidence,
    \item structured retention report,
    \item follow-up conversation history.
\end{itemize}

\subsection{Retrieval-Augmented Generation}
The knowledge base contains telecom-specific playbooks for:
\begin{itemize}
    \item pricing retention,
    \item contract upgrades,
    \item support recovery,
    \item payment friction,
    \item digital adoption,
    \item high-value customer handling.
\end{itemize}

Documents are embedded using \texttt{gemini-embedding-001} and indexed locally with FAISS. This keeps the system compatible with free-tier hosting constraints.

\subsection{Structured Output}
The retention strategist returns a report with:
\begin{itemize}
    \item business context,
    \item risk summary,
    \item key drivers,
    \item retrieved evidence,
    \item recommended actions,
    \item priority order,
    \item next-touch plan,
    \item confidence notes.
\end{itemize}

\section{Architecture Summary}
\begin{enumerate}
    \item User submits a customer profile or CSV file.
    \item Shared inference loads the trained ML artifacts and computes churn probability.
    \item The app extracts major churn drivers for explainability.
    \item The LangGraph workflow formulates a retrieval query.
    \item FAISS returns relevant retention playbooks.
    \item Gemini generates a structured retention plan grounded in the retrieved evidence.
    \item The user can ask follow-up questions inside the same case session.
\end{enumerate}

\section{Deployment}
The final system is deployed as a hosted Gradio application on Hugging Face Spaces. The deployment uses free-tier compatible components:
\begin{itemize}
    \item Gradio for the interface,
    \item scikit-learn for ML inference,
    \item LangGraph for agent orchestration,
    \item FAISS for local retrieval,
    \item Gemini API for structured report generation.
\end{itemize}

The app reads \texttt{GEMINI\_API\_KEY} from Space secrets. When model artifacts are absent in a deployment branch, the application can bootstrap training automatically before serving predictions.

\section{Conclusion}

The resulting system satisfies both the predictive ML requirements and the agentic AI retention strategy requirements of the project.
